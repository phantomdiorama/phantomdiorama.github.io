My blog setup is super minimal. It’s just a single script that wraps around Pandoc. Ok there’s a second script that does some formatting checks. But that’s it. I call this setup CodNap. That’s Pandoc backwards. I thought it was funny.

Obviously, it’s not as automated as an actual static site generator like Hugo, but it is less complicated–with so few moving parts, I’m able to keep the whole thing in my head. And I can keep everything in a simple two-folder deep directory tree. The root folder is called blog and is kept in Dropbox for easy backup and sync.

Building the site is just a matter of running two scripts on that folder…

check

This script, mysteriously called check, checks for any outstanding to dos, outstanding typos (with typos), or markdown formatting issues (with markdownlint). I could probably do without this if I was more attentive. I’m not.

build

Build iterates over the list of markdown files in blog, uses Pandoc to convert them to html, and then builds a copy of the site in blog/public.

push

After manually checking the links in Firefox, the contents of blog/public gets copied into a git repo which, when pushed, ends up on github pages).

wrapping up

I started off saying my setup was minimal. While it’s true it involves minimal tooling, there are a few areas which I could streamline—I hand edit the atom feed, for one. And automating image optimisation would be a nice edition. But change is bad and it should feel bad.
