how i do this

2024–02-26

My blog setup is super minimal. It’s just a single script that wraps around Pandoc. Ok there’s a second script that does some pre checks. But that’s it. I call this setup codnap. That’s Pandoc backwards. The github repo needed a name, ok?

Obviously, it’s not as automated as actual Static Site Generators like Jekyll or Hugo, but it is simpler, more flexible and allows me to keep all my blogging bits and pieces (markdown, images, css, everything) in a single folder instead of a thicket of sub-directories. And building is just a matter of running two scripts on that folder…

check

This does pre-build checks for outstanding todos (w/ ripgrep), outstanding typos (w/ typos) and markdown errors (w/ markdownlint). On its face this sounds less minimal than using the eyes attached to that face, but remember behind that face is a skull of laziness. It’s fine.

build

This uses Pandoc to make html from markdown and moves the resulting files into /public. Images get moved to /public/images.
