tools

First, what I didn’t choose. I didn’t want to drive into the various self-hosted pocket alternatives like ArchiveBox or Wallabag as they are large and too complicated for my needs. I also didn’t want to pick a browser addon like SingleFile as they cannot be automated[^A cli version of SingleFile exists but still requires running a browser].

So what’s left? There’s wget and curl. Both tools have been around forever and a designed for this kind of usecase. Another older tool is Pandoc, a document converter which can grab webpages not just as html but dozens of other formats as well. Monolith is relatively new, but it’s main use is bundling entire pages—including images, media, css, JavaScript—into a single html5 file. Lastly, there’s readibility-cli, an installable version of Firefox’s reader mode, which strips the cruft from a website and saves clean html. For testing purposes I’m using shed an ease-of-use bash script which calls readability.

testing

When testing I didn’t try anything fancy just used each tool to grab three random pages (from Wikipedia, ArsTechnica and the BBC), measured how long they took, and how big the resulting file were. The script I used is on the here.
